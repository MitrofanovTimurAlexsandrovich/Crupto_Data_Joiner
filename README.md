# 📥 Crypto Data Joiner

Инструмент для автоматической загрузки исторических данных фьючерсных USDT-пар с биржи Bybit и их объединения в единый готовый датасет для дальнейшего анализа или бэктестинга.

---

## 📦 Структура проекта

```
Crypto_Data_Joiner/
├── Bybit_Futures_USDT_download.py   # Загрузка сырых данных с Bybit
├── Prepare_data.py                  # Обработка и объединение данных
├── Raw_Data/                        # Папка для сырых загруженных файлов
├── stats/                           # Статистика обработки
├── logs/                            # Логи выполнения
└── trade_data_processing.log        # Основной лог обработки
```

---

## ⚙️ Что делает проект

### 1. `Bybit_Futures_USDT_download.py`
- Подключается к публичному API Bybit
- Скачивает исторические свечные данные (OHLCV) по фьючерсным USDT-парам
- Сохраняет сырые файлы в папку `Raw_Data/`

### 2. `Prepare_data.py`
- Читает сырые файлы из `Raw_Data/`
- Очищает и нормализует данные (типы, форматы timestamp и т.д.)
- Объединяет (join) несколько файлов/периодов в один непрерывный датасет
- Сохраняет результат, пригодный для бэктестинга (например, для GRIDBOT)
- Ведёт подробные логи обработки

## 🚀 Установка

```bash
pip install -r requirements.txt
```

> Убедитесь, что установлены `pandas`, `requests` и `tqdm` (или их эквиваленты, указанные в requirements).

---

## 🖥️ Использование

### Шаг 1 — Загрузка данных

```bash
python Bybit_Futures_USDT_download.py
```

Скрипт скачает исторические данные с Bybit и сложит их в `Raw_Data/`.

### Шаг 2 — Подготовка и объединение

```bash
python Prepare_data.py
```

Скрипт обработает все файлы из `Raw_Data/`, объединит их и сохранит итоговый CSV-файл.

---

## 📁 Формат выходных данных

Итоговый CSV содержит стандартные OHLCV-колонки:

```csv
timestamp,open,high,low,close,volume
2020-01-01 00:00:00,7195.5,7196.0,7194.5,7195.0,1234.5
...
```

Этот формат напрямую совместим с **[GRIDBOT](https://github.com/MitrofanovTimurAlexsandrovich/GRIDBOT)** и другими торговыми инструментами, принимающими стандартный OHLCV CSV.

---

## 📋 Логирование

В процессе работы ведётся лог в файл `trade_data_processing.log` и папку `logs/`. Там можно отследить:

- какие файлы были обработаны
- количество строк до и после очистки
- обнаруженные пропуски или дубликаты
- итоговый диапазон дат датасета
